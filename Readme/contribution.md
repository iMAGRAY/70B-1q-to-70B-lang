# Contribution Guide

Мы приветствуем pull-requests!

## Как начать

1. Откройте issue с описанием фичи/баг-репорта.
2. Согласуйте объём работ с core-командой.
3. Сделайте `fork`, создайте ветку `feature/<topic>`.

## Код ревью

* PR должен проходить CI (pytest + linters).
* Изменения >300 строк требуют как минимум 1 reviewer из core.
* Обязательно описание **что** и **почему** меняется.

## Коммиты

* Используйте Conventional Commits: `feat:`, `fix:`, `docs:`, `refactor:`, etc.
* Один логический блок → один коммит.

## Чек-лист перед PR

- [ ] `pytest -q` успешен
- [ ] `black sigla/` запущен
- [ ] `flake8 sigla/` без ошибок
- [ ] Документация обновлена (при изменении API)

## Лицензия

Согласуясь с MIT, контрибуторы предоставляют право включать свой код в репозиторий под той же лицензией.
